About the Project

Since ages throwing a ball has been one of the biggest sports in the world. Baseball, handball and American football. But taking it one step further, Color Splat forces the player to be both strategic and work out to hit the evil characters trying to break into your fortress.

The game combines both reality and the virtual world when the player must use a physical ball, to throw towards the screen, which then continues into the unreality. To then reload the player needs to get the ball - wherever it landed. This game is a creative game changer making the player solve the old throw-a-ball game in a completely different way.

Give it a try and we promise that you will both laugh and do your daily workout.
Peace out!

Technologies and techniques

This project has been developed under the version 5.2 of Unity Personal Edition.

Augmented virtuality

In our set-up, the player is placed approximately 2 meters away from the middle of the screen. We take average height as a basis of our calculations, but children can also play without having a different feeling. Thus, we have two extremities of the direction vector of the physical ball, embodied by the average height of the people for the initial position, and the position where the ball has touched the screen for the final position.


The position of the physical ball on the screen is converted to a position in the virtual world, and a virtual ball is created at this location with a Unity Rigidbody. We scale the vector created above and we add a force to this virtual ball. Its position in-game is then calculated by the Unity physics engine. This provides a realistic parabolic shot that simulates the course that the ball would have had if it had continued behind the screen.

An interesting thing is the concept of hitting objects on a screen. A new player is most likely to try to hit the ball on the spot where the character is seen, without taking the 3D game world into account, though most adapt when they realize this after two or three throws. The concept taking the game world into account when throwing the ball would most likely have been easier to grasp if we had implemented stereoscopic vision and made the virtual space part of the real world.

Hardware and computer vision

This project was designed to be played on a 2 by 4 meters 4K screen in the Visualisation Studio VIC. Behind the screen lays a powerful projector, which displays the virtual world, and a simple camera filming the back of the screen. Every object on the player side of the screen casts a shadow that is filmed by the camera. That is why computer vision is needed to detect shadow events.


The computer vision software we used is Community Core Vision. The software substracts the current video frame from the background frame, thus returning a blob, representing the shadow event. To spawn a paintball in our game, we only need the position of this event in the filmed space, and its session id at a given time, to avoid duplicates. These information can be sent to a network via several different protocoles, like UDP or TCP. However, we needed TUIO OSC messages to send to Unity, therefore the use of the stable version 1.2 of CCV, instead of the latest 1.5 version which does not feature TUIO output.

In Unity, we chose C# as the scripting language, so we chose the associated TUIO11_NET plugin. It provides a simple library with a TUIO client, listener, and several types of objects, like cursors, which have an id and a position. With the help of this plugin, we created a script that gets any new event, and sends its coordinates to the Unity game. From there, we can calculate the parabolic shot as explained previously.
